## 注意事项

1. **首次推送**：由于是空仓库，使用 `--force` 参数强制推送
2. **中文编码**：确保命令行使用UTF-8编码（bat脚本已设置）
3. **Git配置**：确保已配置用户名和邮箱：
   ```cmd
   git config --global user.name "你的名字"
   git config --global user.email "你的邮箱"
   ```

4. **认证问题**：如果遇到认证问题，可能需要配置GitHub的Personal Access Token

## 问题排查

### 如果推送失败

1. **认证失败**：
   - 检查GitHub账号登录状态
   - 配置Personal Access Token
   - 使用SSH密钥

2. **网络问题**：
   - 检查网络连接
   - 尝试使用VPN
   - 使用GitHub镜像

3. **仓库权限**：
   - 确认有写入权限
   - 检查仓库设置

4. **冲突问题**：
   - 使用 `--force` 强制推送（首次推送可以使用）
   - 或者先 `git pull` 再推送
