# 文件管理 / 数智平台 — 迭代优化建议

> 基于当前实现抽象出的可优化方向，按**架构 / 数据 / 体验 / 性能 / 功能 / 维护**分类，便于排期迭代。

---

## 一、架构与工程化

| 方向 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| **页面与逻辑解耦** | 文档模块的 4 个页面（绩效文档、部门详情、AI 搜索、全部文件）的 HTML 以巨大模板字符串写在 `main.js` 的 `pageData` 里，修改易出错、难复用。 | 将每个页面的 HTML 抽成独立模板文件（如 `doc-performance.html` 片段）或至少抽到 `scripts/doc-pages.js`，用 `fetch` 或构建时注入；或引入轻量模板引擎（如 原生 `template` + 填空）。 | 高 |
| **文档模块独立化** | 文档相关逻辑（约 16280 行起）与主应用混在同一 `main.js`，文件过长（约 1.7 万行）。 | 将文档模块拆成独立 JS 模块（如 `doc-module.js`），通过 `type="module"` 或构建打包引入，便于按需加载与维护。 | 高 |
| **路由与状态** | 用 `loadPage(name, param)` + `currentPage` + 多处 `window._doc*` 全局变量驱动页面与列表。 | 引入轻量路由（如基于 hash 的简单 router）和单一文档状态对象（如 `docState`），避免分散的 `_doc*`，便于调试与扩展。 | 中 |
| **构建与资源** | 无构建流程，直接引用 CDN（Tailwind、Font Awesome 等）。 | 若后续要压缩、按需加载或多环境配置，可引入 Vite/Webpack 等，并统一管理 CDN 与本地资源。 | 低 |

## 二、数据与状态

| 方向 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| **Mock 数据集中管理** | `docMockData`、部门颜色/图标等写在 `main.js` 中，树结构在 `renderDocAllFileTree` 里硬编码。 | 将 mock 数据移到独立 JSON/JS 配置文件（如 `data/doc-mock.js`），树结构可配置化（如 `defaultCollapsed`、层级、图标），便于切换环境或接入真实 API。 | 高 |
| **状态持久化** | 最近使用、搜索历史等仅存内存，刷新即丢失。 | 对「最近使用」「搜索历史」等使用 `localStorage` 做持久化，并约定 key 前缀（如 `doc_recent_`）便于后续清理或多端同步。 | 中 |
| **列表与树数据源统一** | 部门列表、全部文件树、搜索结果是多处对 `docMockData` 的 filter，逻辑分散。 | 抽象一层「文档数据服务」（如 `getDocsByDeptMonth`、`getDocsByQuery`、`getTreeFromDocs`），列表/树/搜索都通过该层取数，便于以后换成接口。 | 中 |

## 三、体验与交互

| 方向 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| **预览与列表联动** | 预览浮层支持上一份/下一份，依赖 `_docPreviewList` 顺序；从树或部门列表进入时需保证该列表与当前视图一致。 | 已在设置 `_docPreviewList`，可再核对：从部门列表、全部文件树、搜索/联想进入预览时，列表是否均为「当前视图下的文档顺序」；必要时为预览增加「当前第 x / 共 y 份」提示。 | 中 |
| **树节点与面包屑联动** | 全部文件下面包屑目前仅「全部文件」；树点击后未更新面包屑路径。 | 点击树节点时更新面包屑为「全部文件 > 月度绩效归档 > 2026年 > 03月」等形式，并支持点击面包屑某一级快速折叠/定位。 | 中 |
| **空状态与错误态** | 已有空状态、骨架屏；错误态（如请求失败、权限不足）多为 alert/notification。 | 列表/树区域增加统一的「加载失败，点击重试」、无权限时的占位文案与操作引导，减少裸 alert。 | 低 |
| **键盘与无障碍** | 搜索框支持 Enter 发送；树节点、列表项主要靠点击。 | 为树节点、列表项增加键盘导航（上下键、Enter 展开/打开）；为图标按钮补充 `aria-label`，浮层获得焦点时 trap focus，便于无障碍与键盘用户。 | 低 |

## 四、性能与可访问性

| 方向 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| **大列表渲染** | 部门文档列表分页加载更多，单次渲染条数可控；全部文件搜索结果一次渲染。 | 若搜索结果很多，可对结果列表也做分页或虚拟滚动，避免一次渲染数百个 DOM。 | 中 |
| **树节点按需渲染** | 当前树是一次性拼 HTML 字符串，所有节点（含默认折叠下的子节点）都会出现在 DOM 中。 | 可改为：默认折叠的子树不渲染 DOM，点击展开时再插入子节点（懒加载子树），减少首屏 DOM 数量，尤其适合层级深、节点多的场景。 | 低 |
| **图标加载** | Font Awesome 已用 CDN + preconnect，图标多时仍可能有一闪。 | 对首屏关键图标考虑内联 SVG 或单独小体积图标集，或保留 FA 但加 `font-display: optional` 与占位，减少布局偏移。 | 低 |

## 五、功能扩展（与现有能力衔接）

| 方向 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| **上传 / 新建 / 权限** | 目前为「演示」提示，无真实接口。 | 定义与后端约定好的接口形态（上传 multipart、新建文件夹 payload、权限列表/更新），前端先做表单与调用占位，再对接真实 API。 | 中 |
| **收藏与最近使用** | 最近使用来自预览记录；「我的收藏」无数据与操作。 | 为文档项增加「收藏」状态（mock 可先放 localStorage），在列表/树中展示收藏图标与筛选入口，与「最近使用」并列。 | 中 |
| **分享** | 分享为「链接已复制」演示。 | 若需真实分享：可扩展为生成带权限的分享链接、过期时间、权限级别（只读/下载），并在前端展示「已分享」状态。 | 低 |
| **AI 搜索** | 当前为前端关键词 filter 模拟。 | 接入真实检索/向量搜索 API 后，将 `sendDocAiQuery` 改为请求接口，保留现有气泡与结果卡片 UI，增加加载态与错误态。 | 中（视后端进度） |

## 六、文档与维护

| 方向 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| **文档模块说明** | 文档逻辑嵌在巨大 main.js 中，新人难以快速定位。 | 在仓库中增加 `docs/文档模块说明.md`：入口页、路由与页面对应关系、主要全局变量与数据流、mock 与接口预留点、常见修改点（如树默认展开、列表字段）。 | 高 |
| **关键交互约定** | 树展开/折叠、列表多选、预览上一份/下一份等依赖 DOM 结构和 class 命名。 | 对关键节点（如 `.doc-tree-node`、`.doc-tree-children`、`.doc-batch-bar`）在代码或文档中注明「勿随意改 class/结构，否则 xx 功能会失效」，避免误伤。 | 中 |
| **版本与变更** | 无统一变更记录。 | 用 `CHANGELOG.md` 或项目内「迭代记录」记录每次发布的功能/体验/技术债变更，便于回溯与排期。 | 低 |

---

## 建议实施顺序（概要）

1. **短期（立刻可做）**  
   - Mock 数据与树配置抽离到独立文件。  
   - 最近使用/搜索历史 `localStorage` 持久化。  
   - 补充「文档模块说明」和关键 DOM 约定说明。

2. **中期（小步迭代）**  
   - 页面模板从 `main.js` 抽离；文档相关逻辑拆成独立模块。  
   - 统一文档数据访问层（如 `getDocsByDeptMonth`、`getDocsByQuery`）。  
   - 树/列表与面包屑联动；预览列表顺序与「当前视图」一致。

3. **长期（按需推进）**  
   - 轻量路由 + 单一文档状态对象。  
   - 树懒加载子树、列表/搜索结果虚拟滚动或分页。  
   - 上传/新建/权限/分享/AI 搜索与真实接口对接。
